# 帮助手册

## 部署发布项目

每次git pull后都有很多冲突，解决后部署不生效，浪费很多时间

#### 方法

* 在dev分支开发
* build && deploy && push

每次deploy就完事了，会把更新的打包代码发到master分支上

* github中更改默认分支为dev
* 删除master分支
* 在dev分支上新建master分支，push
* github中更改默认分支为master
* 浏览效果
* 切换dev分支开发
